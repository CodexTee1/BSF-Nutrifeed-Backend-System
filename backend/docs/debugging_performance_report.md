# Debugging and Performance Report

During development, the main issues encountered were environment mismatch, database schema drift, request-format errors, and unstable local server behavior. The first major problem was that different Python environments were being used interchangeably, which caused missing-package errors for modules such as `flask_jwt_extended`. This was resolved by standardizing execution around one working virtual environment. Another issue appeared after the data models were expanded: the SQLite database no longer matched the updated schema, leading to errors such as missing columns. This was fixed by generating and applying a fresh Flask-Migrate migration and recreating the local database correctly.

During API testing, Postman initially returned `415 Unsupported Media Type` because JSON payloads were not being sent with the correct request body configuration. This was solved by setting requests to `raw` and `JSON`, then validating the endpoints one by one. A separate issue involved Flask’s development reloader, which made the server appear to stop unexpectedly during testing. To improve reliability, the app was run without the reloader for local verification.

Several improvements were made to enhance performance and stability. These included JWT protection for sensitive routes, role-based access control for admin actions, validation of required input fields, pagination on list endpoints, and lightweight JSON responses. Together, these changes made the backend more reliable, easier to debug, and better prepared for demonstration and future scaling.
